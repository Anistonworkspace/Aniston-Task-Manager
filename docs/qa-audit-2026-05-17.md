### §6 Service-layer tests

**Architectural concern**: 68 of 89 server test files use `jest.mock` on
the models layer. This is a §6.1 anti-pattern at scale. The skill says
"prefer integration tests with a real (test) database over fully-mocked
unit tests". CLAUDE.md documents this as intentional ("All mock-DB; no real
Postgres") but it costs the project:
- Schema drift can't be detected by tests
- Real query bugs (N+1, slow lookups, transaction issues) invisible
- Concurrency / race conditions never exercised
- Mock-vs-real divergence accumulates silently

**Recommendation** (per §6.3): introduce a parallel **integration test
suite** that runs against a real test Postgres (via Docker / testcontainers)
with per-test transaction rollback. Start with the 5 most critical
services: `permissionEngine`, `dependencyService`, `approvalLifecycleService`,
`recurringTaskService`, `notificationService`.
